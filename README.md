# TFG_GemmaBeatrizVate
IoT network traffic classification with JA4+ fingerprints

## Environment

### Install `virtualenv`

```bash
sudo apt-get update
sudo apt-get install python3-virtualenv
```

### Creating virtual environment

```bash
virtualenv venv
```

### Activate env

```bash
source venv/bin/activate
```

### Install dependencies

```bash
pip install -r requirements.txt
```

## Project Workflow

## STEP 1: Extracting logs and fingerprints with Zeek

The FoxIO repository is used to generate the fingerprints. In this phase, `Zeek` is used because it is faster than Python. The only fingerprint it does not generate directly is `JA4X`.

```bash
cd /mnt/d/TFG_GemmaBeatrizVate
mkdir -p /mnt/d/TFG_GemmaBeatrizVate/1_Data/zeek_logs
zeek -r 1_Data/Original_Pcaps/BenignTraffic.pcap \
  0_Init/ja4/zeek/__load__.zeek \
  Log::default_logdir=/mnt/d/TFG_GemmaBeatrizVate/1_Data/zeek_logs/logs
```

Files generated by Zeek:

- `JA4` -> `ssl.log`
- `JA4S` -> `ssl.log`
- `JA4X` -> `x509.log`, but Zeek does not save JA4X fingerprints directly.
- `JA4H` -> `http.log`
- `JA4L` -> `conn.log`
- `JA4LS` -> `conn.log`
- `JA4T` -> `conn.log`
- `JA4TS` -> `conn.log`

For alternative scenarios or traffic, the same process is repeated by changing the input PCAP and the output log directory.

## STEP 2: Consolidating fingerprints into a single CSV and creating JA4X

The main script for this phase is:

```bash
python 2_Development/2_1_Preprocessing/combine_zeek_fingerprints.py
```

This script combines the logs generated by Zeek into a single connection dataset containing JA4+ fingerprints and the associated IoT device information.

The `JA4X` extraction was implemented using the FoxIO Python `ja4x.py` script as a reference, adapting its logic to calculate the fingerprint from the certificate information available in Zeek's `x509.log`: [FoxIO JA4X Python script](https://github.com/FoxIO-LLC/ja4/blob/main/python/ja4x.py).

Files and modules used:

- `combine_zeek_fingerprints/config.py`: defines the input and output paths, the source PCAP, the Zeek log directory, and the `MAC_TO_DEVICE` and `MAC_TO_CATEGORY` dictionaries.
- `combine_zeek_fingerprints/reader.py`: reads Zeek logs and extracts the `IP -> MAC` map from the PCAP using `tshark`.
- `combine_zeek_fingerprints/processor.py`: normalizes `conn.log`, `http.log`, and `ssl.log`, assigns devices, and joins the information by `connection_id`.
- `combine_zeek_fingerprints/ja4x.py`: calculates the `JA4X` fingerprint from the certificates in `x509.log`.

Main inputs:

- `1_Data/Original_Pcaps/BenignTraffic.pcap`
- `1_Data/zeek_logs/logs/conn.log`
- `1_Data/zeek_logs/logs/http.log`
- `1_Data/zeek_logs/logs/ssl.log`
- `1_Data/zeek_logs/logs/x509.log`

Summary of the process:

1. Extracts the real MAC addresses from the PCAP.
2. Reads the Zeek logs.
3. Normalizes the main connection, HTTP, and TLS columns.
4. Calculates `JA4X` from `x509.log` and links it to TLS connections using the certificate fingerprints available in `ssl.log`.
5. Assigns the device name and category from the MAC address.
6. Generates a final CSV by combining the connections from `conn.log` with the available information from `http.log` and `ssl.log`.

Output:

- `1_Data/zeek_logs/combined_fingerprints.csv`

The execution proof is saved in:

- `1_Data/zeek_logs/combined_fingerprints.txt`

## STEP 3: Extracting raw features from fingerprints

The main script for this phase is:

```bash
python 2_Development/2_1_Preprocessing/extract_raw_features.py
```

This script takes the combined CSV generated in the previous step and splits the JA4+ fingerprints into simpler columns for later analysis or model training.

The fingerprint parameters extracted in this step are based on the JA4+ format described by FoxIO in [JA4+ Network Fingerprinting](https://blog.foxio.io/ja4+-network-fingerprinting).

Files and modules used:

- `extract_raw_features/config.py`: defines the input file, output file, base columns, and expected fingerprint columns.
- `extract_raw_features/processor.py`: loads the CSV, calculates fingerprint coverage, converts timestamps, applies the parsers, and saves the results.
- `extract_raw_features/parsers.py`: contains the functions that split each fingerprint into its components.
- `extract_raw_features/__init__.py`: exposes the package functions so they can be imported by the main script.

Main input:

- `1_Data/zeek_logs/combined_fingerprints.csv`

Summary of the process:

1. Reads the combined CSV.
2. Checks which fingerprint columns are available.
3. Converts Zeek timestamps into a readable format.
4. Extracts components from the fingerprints currently configured in `processor.py`: `JA4`, `JA4S`, `JA4X`, `JA4T`, `JA4TS`, and `SNI`.
5. Builds the final dataset with identification columns, original fingerprints, and extracted features.
6. Also saves individual files by fingerprint type when applicable.

Outputs:

- `1_Data/raw_fingerprints/raw_features.csv`
- `1_Data/raw_fingerprints/raw_features_clean.csv`, if complete rows without null values exist.
- Individual files in `1_Data/raw_fingerprints/`, for example `JA4_raw.csv`, `JA4S_raw.csv`, `JA4X_raw.csv`, `JA4T_raw.csv`, `JA4TS_raw.csv`, and `SNI_raw.csv`.

The execution proof is saved in:

- `1_Data/raw_fingerprints/raw_features.txt`
